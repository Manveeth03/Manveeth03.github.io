Dynamic Programming:
The Art of Remembering

How a simple principle — "never solve the same problem twice" — unlocks solutions to some of the hardest challenges in computer science.

✦ SLA Activity 1 ✦ April 2026

Introduction

Imagine you are asked to calculate the 40th Fibonacci number by hand, using only the rule that each number is the sum of the two before it. You would be repeating calculations thousands of times — computing the same sub-results over and over without ever remembering what you had already found. Now imagine keeping a small notebook where you write down every result the moment you calculate it. Suddenly, the problem that once felt endless becomes trivial.

This is the essence of Dynamic Programming (DP) — one of the most powerful and elegant paradigms in algorithm design. First formalized by Richard Bellman in the 1950s, DP is not a single algorithm but a strategy: break a complex problem into simpler overlapping subproblems, solve each subproblem once, store its result, and build up to the final answer. It sits at the heart of solutions to problems in route optimization, genomics, finance, game AI, and beyond.

"Those who cannot remember the past are condemned to repeat it." — George Santayana. In algorithms, those who cannot remember subproblem results are condemned to recompute them.

When Does Dynamic Programming Apply?

Not every problem can — or should — be solved with DP. A problem is a candidate for this technique when it exhibits two key properties:

The Two Pillars of DP
  • Optimal Substructure: The optimal solution to the whole problem can be constructed from optimal solutions to its subproblems. For example, the shortest path from city A to city C through city B is composed of the shortest path A→B and the shortest path B→C.
  • Overlapping Subproblems: The same smaller problems recur many times during computation. This is what makes plain recursion expensive and makes memoization so valuable.

If a problem has both properties, DP can reduce what might be exponential brute-force time into polynomial time — a dramatic improvement that is the difference between a program running in milliseconds versus centuries.

Two Approaches: Top-Down vs Bottom-Up

There are two classic ways to implement dynamic programming, each with its own character.

Top-Down: Memoization

Here, you write a recursive function exactly as you would naturally think about the problem, but add a cache (a dictionary or array) to store results. Before doing any work, you check: "Have I solved this subproblem before?" If yes, return the stored answer immediately. This approach is intuitive and only solves the subproblems that are actually needed.

# Top-Down DP — Fibonacci with memoization
def fib(n, memo={}):
    if n <= 1:
        return n
    if n not in memo:
        memo[n] = fib(n-1) + fib(n-2)
    return memo[n]

# fib(40) now runs in O(n) time instead of O(2ⁿ)

Bottom-Up: Tabulation

Here, you start from the smallest subproblems and iteratively fill in a table, working your way up to the final answer. There is no recursion — only loops. This is often faster in practice due to lower overhead and better cache performance.

# Bottom-Up DP — Fibonacci with tabulation
def fib_tab(n):
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

Classic DP Problems You Should Know

Dynamic programming shines across a wide range of problem types. Some of the most important examples include:

0/1 Knapsack Problem: Given a set of items with weights and values, and a bag of fixed capacity, maximize the total value you can carry. Each item can either be taken or left — no partial items allowed. DP allows you to systematically evaluate every combination in O(n × W) time, far better than the brute-force O(2ⁿ).

Longest Common Subsequence (LCS): Given two strings, find the longest sequence of characters that appears in both, in order. This underpins the diff tool in version control systems like Git, and is used in DNA sequence analysis in bioinformatics.

Matrix Chain Multiplication: Given a sequence of matrices to multiply, find the parenthesization that minimizes the total number of scalar multiplications. The naive approach has exponential possibilities; DP solves it in O(n³).

Shortest Path (Bellman-Ford / Floyd-Warshall): Finding the shortest path in a weighted graph — including graphs with negative edge weights — is solved elegantly with DP recurrences.

Real-World Applications

Dynamic programming is not confined to textbooks. It quietly powers many systems we use daily. Spell checkers use DP to compute edit distance — the minimum number of insertions, deletions, or substitutions needed to transform one word into another. Google Maps and GPS navigation systems use DP-based shortest path algorithms to find optimal routes. Auto-correct on your phone, protein folding simulations in medicine, speech recognition, and even resource allocation in cloud computing all rely on the same fundamental idea.

In competitive programming and technical interviews, DP problems are famously challenging because they require both recognizing the subproblem structure and defining the correct recurrence relation — a skill that only comes with practice.

Conclusion

Dynamic Programming is a testament to a deceptively simple insight: intelligence is about remembering, not just computing. By storing the results of subproblems and building solutions from the ground up, DP transforms problems that would take the age of the universe to brute-force into ones solvable in fractions of a second.

For any student of algorithms, mastering dynamic programming is a rite of passage. It teaches you to see the hidden structure inside problems — to ask not just "what is the answer?" but "what smaller answers does this answer depend on?" That shift in thinking is the true gift of this technique.

Start with Fibonacci. Move to Knapsack. Then tackle LCS. Each problem will sharpen your intuition, and soon you will find yourself recognizing the DP structure in problems you once thought were unsolvable. The notebook is open — start filling it.
